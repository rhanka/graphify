# SPEC ‚Äî Git-flow view: branch-label legibility policy

> Status: **RATIFIED (b) ‚Äî phases 1+2 together** ¬∑ Principal decision 2026-07-05, after an
> Opus 4.8 + Codex (5.5xhigh) double-consensus brainstorm (options dossier presented in
> writing, nothing pre-applied). Companion to the git-flow renderer lot
> (`packages/graph`: `layout-gitflow.ts`, flow-port edge styles, `git-flow` layout id).
> The shipped view itself (grammar, edge styles, layout API, pipeline) is specified in
> [SPEC_GITFLOW_VIEW.md](SPEC_GITFLOW_VIEW.md).
>
> Problem (principal's verdict on real data): with dozens‚Äìhundreds of visible branches ‚Äî
> including many autogenerated `worktree-agent-<hash>` ‚Äî the branch-name pills overlap into
> unreadable soup ("le nom des branches prend trop de place, ou bien il n'y a pas d'espace").
> Shared diagnosis: this is a **label-policy** problem, not a geometry problem.

## Invariants (hard)

- **I1 ‚Äî Zero overlap**: no two rendered label pills may overlap at any zoom (MapLibre-style
  hard rule). Culling is the mechanism; overlap is never "allowed a little".
- **I2 ‚Äî Structure always visible**: culling/demotion affects **names only** ‚Äî branch lanes,
  commits and connectors always render. Autogenerated branches remain visible as structure.
- **I3 ‚Äî Full name on demand**: hovered / selected / search-matched branches always show their
  **full, uncompacted** name (top priority, never culled).
- **I4 ‚Äî CPU-side policy**: implemented as a label pre-pass (compaction ‚Üí priority ‚Üí placement)
  before the existing `boxDimensions`/overlay draw path. No shader work; no scene-contract or
  layout-signature change. Every brick is independently toggleable.
- **I5 ‚Äî Rejected anti-pattern**: never widen lanes/`laneGap` to make room for labels.

## Phase 1 (ship together)

### P1.1 Semantic name compaction + autogenerated demotion
- Strip no-information prefixes: `refs/heads/`, `origin/`.
- Compact conventional prefixes: `feature/`‚Üí`feat:`, `feat/`‚Üí`feat:`, `fix/`‚Üí`fix:`,
  `chore/`‚Üí`chore:`, `hotfix/`‚Üí`hotfix:`, `release/`‚Üí`rel:`.
- Autogenerated patterns (configurable list, defaults):
  - `worktree-agent-<hash>` ‚Üí `agent:<7hex>`;
  - `dependabot/<eco>/<pkg>-<ver>` ‚Üí `deps:<pkg>`;
  - **demoted class**: autogenerated branches get NO persistent pill by default ‚Äî names are
    hover/selection-first (I2 keeps their geometry visible).
- Width budget: middle-ellipsis beyond ~22 chars, preserving the meaningful SUFFIX
  (`‚Ä¶-geo-integration`, not `feat/cadrage-geo-‚Ä¶`).
- Ambiguity guard: if two compacted names collide on the same string, disambiguate with a
  short hash suffix.

### P1.2 Priority collision culling (MapLibre-style)
- Screen-space AABBs from the measured pill boxes; grid occupancy; place in priority order,
  drop (don't draw) anything that would overlap an accepted box.
- Priority (descending): hovered/selected/search-matched ‚Üí trunk labels (`main`/`master`/
  `develop`, one per repo band) ‚Üí release/hotfix ‚Üí human feature branches (recency-weighted:
  newer tip = higher) ‚Üí tip-only refs ‚Üí autogenerated (only present at all in T2, see P2.1).
- Determinism: stable sort key (priority, then branch-name hash) so the same camera always
  yields the same label set; hysteresis margin (~8 px) so panning doesn't flicker labels.

### P1.3 Hover / selection / search full labels
- The interaction layer (studio) can mark branches hovered/selected/searched; those labels
  render full-name, top-priority, uncompacted (I3). Hit targets cover the branch lane
  interval and tip, not just the label.

## Phase 2 (ship together with Phase 1 ‚Äî principal decision (b))

### P2.1 Zoom LOD tiers (with hysteresis)
- **T0 (far)**: trunk + selected/hovered only.
- **T1 (mid)**: + human-named branches that survive P1.2 culling.
- **T2 (near)**: + autogenerated branches (compacted `agent:<hash>` form), still culling-gated.
- Tier thresholds on camera zoom with ¬±10% hysteresis (no popping storms while zooming).

### P2.2 Anchor fallback fork ‚Üí tip (bounded)
- Candidate anchors per label: fork-left (default) ‚Üí tip-right (fallback when the fork
  region is saturated) ‚Üí ONE bounded stagger slot (small vertical offset). No unlimited
  stacking. A staggered/tip-anchored label must keep a visual tie (its colour) to its lane.

## Non-goals / later

- **Gutter "branch directory" mode** (fixed side column, leader lines): alternate mode for
  dense-repo archaeology, opt-in, NOT the default ‚Äî separate lot.
- Numeric label scoring beyond recency (e.g. activity heat) ‚Äî later, additive.

## Acceptance criteria

- AC1: dense synthetic golden (‚â•40 branches incl. ‚â•15 `worktree-agent-*`) renders with
  **zero overlapping pills** at 3 zoom levels (T0/T1/T2 probes).
- AC2: on the real 4-repo agent-stats scene, no `worktree-agent-*` persistent pill at T0/T1;
  structure (lanes/commits) unchanged.
- AC3: pre-existing non-gitflow goldens stay byte-identical (label pre-pass is opt-in via the
  git-flow path).
- AC4: hover/selection shows the full name even for a culled/demoted branch.
- AC5: determinism ‚Äî same scene + same camera ‚áí identical label set across runs.
